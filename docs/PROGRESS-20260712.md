# CUACADX Progress — 2026-07-12

## Phase 1 Complete: Real GFS → FourCastNet Pipeline

### Breakthrough: Model Divergence Fixed

**Root cause**: Two bugs combined to produce garbage output from FourCastNet even with
real input data:

1. **HGT unit mismatch** (primary): GFS provides geopotential in gpm (meters), but
   FourCastNet expects specific geopotential (m²/s²). The model was trained on
   `center[z50] = 199,539 m²/s²` but receiving `~20,000 gpm`, translating to a
   normalized value **-30σ below the mean**. This single error poisoned the entire
   auto-regressive forecast. **Fix**: Multiply all HGT-derived variables (z1000, z850,
   z500, z250, z50) by `9.80665` at IC build time (`ingestion/models/fourcastnet_pipeline.py:158`).

2. **dtype mismatch**: `np.load()` returns float64 arrays. The AFNO model weights are
   float32. When the Earth2Studio FCN wrapper does `(x - center) / scale`, the result
   becomes float64 (via broadcasting), then `self.model(x)` passes float64 into the
   AFNO's `Conv2d` which expects float32 weights — causing either a crash or silent
   corruption. **Fix**: Cast IC tensor to `.float()` before passing to model.
